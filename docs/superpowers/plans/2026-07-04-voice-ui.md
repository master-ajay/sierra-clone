## Grounding against the real (not spec-imagined) APIs

- `design-system` exports (same verified API used for Channels UI): `AppShell`,
  `Button`, `Card`, `Table`, `Badge` (`status: success|warning|error|info`, no
  default), `Input`, `Select`, `Modal({ open, onClose, title, children })`,
  `useToast().showToast({ message, variant?, duration? })`, `EmptyState({
  icon?, heading, body, action?: {label, onClick} })`, `MetricCard`.
- Voice backend (verified from `voice/src/voice/routes/*.py` and
  `voice/src/voice/models/*.py`, current on-disk state):
  - `POST/GET/PATCH/DELETE /v1/lines[/{id}]` — `LineResponse = { line_id,
    agent_id, adp_user_id, name, status, line_key, created_at, updated_at }`.
    Admin auth: `X-API-Key`.
  - `POST /v1/lines/{line_id}/calls` (header `X-Line-Key`) → `{ call_id,
    session_id }`.
  - `POST /v1/calls/{call_id}/turns` (header `X-Line-Key`) → `{ reply,
    sentiment: {label, score}, call_sentiment_trend: number[],
    escalation_recommended: boolean }`.
  - `POST /v1/calls/{call_id}/end` (header `X-Line-Key`) → `{
    average_sentiment: number, trend: 'improving'|'declining'|'stable' }`
    (NOT the array shape the original spec draft assumed).
  - `POST /v1/calls/{call_id}/escalate` (header `X-API-Key`) → `{ summary,
    turns }`.
  - `POST /v1/calls/{call_id}/payment` (header `X-API-Key`) →
    `PaymentResponse = { payment_id, call_id, masked_card_last4, amount,
    currency, status, created_at }`, `status` is `'collected'|'blocked'`.
  - Port 8700.
- Agent Studio's agent list: same as Channels UI — `GET /api/agents` on port
  3000 returns `Agent[]` directly, `{ id, name, ... }`.
